FlowFix is a Chrome browser extension that helps you monitor and optimize your Zapier and Make.com workflow integrations. This Privacy Policy describes how FlowFix handles your information. Please read it carefully.

Our Core Promise

FlowFix is 100% local. We do not collect, transmit, or share your personal data with anyone ‚Äî not with us, not with third parties. All processing happens entirely inside your browser.

1. What FlowFix Does

When you visit your Zapier or Make.com dashboard, FlowFix reads the visible information on the page (the "DOM") to extract automation health metrics. It then displays these metrics in a badge overlay on the page and in the extension popup. No data leaves your device.

Metrics FlowFix reads from the page:

Platform Metrics Extracted
Zapier Task usage count, task limit, total zap count, error indicators, paused zap count, plan name
Make.com Operation usage count, operation limit, scenario count, active/inactive/error scenario counts, plan name, team name

FlowFix reads only what is already visible on your dashboard ‚Äî it cannot access your workflow configurations, API keys, connected app credentials, or any data not displayed on the current page.

2. What FlowFix Does NOT Do

  • Send your data to any external server (including our own)
  • Collect personal information such as your name, email, or account details
  • Use analytics, telemetry, or crash-reporting services
  • Intercept or modify your network traffic
  • Access API keys, session tokens, cookies, or authentication credentials
  • Modify your Zapier or Make.com workflows or settings
  • Operate on websites other than zapier.com and *.make.com
  • Load external JavaScript libraries or resources from CDNs
  • Share data with advertisers, data brokers, or any third party
  • Operate when you are not on a supported dashboard page

6. Security Architecture

  • Shadow DOM isolation: The FlowFix health badge is injected using the Shadow DOM API with mode: 'closed', preventing the host page's CSS or JavaScript from reading or modifying the badge element.
  • No innerHTML with external data: All DOM elements are constructed using document.createElement and textContent ‚Äî never innerHTML with untrusted content ‚Äî eliminating XSS injection vectors.
  • Manifest V3: FlowFix is built on Chrome's latest Manifest V3 architecture, which uses service workers instead of persistent background pages and enforces a strict Content Security Policy (script-src 'self') that prevents dynamic code execution.
  • No remote code: FlowFix loads no external scripts, stylesheets, or fonts. All code ships with the extension package.
  • No eval: FlowFix does not use eval(), new Function(), or any other dynamic code execution mechanism.
  • Numeric-only extraction: All values extracted from the page DOM are parsed as numbers using parseFloat() before storage or display, preventing any HTML or script injection from extracted text.
